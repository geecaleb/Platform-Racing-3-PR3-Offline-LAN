# Platform-Racing-3-PR3-Offline-Lan

Platform Racing 3 project aimed at making the game offline playable.

The game is currently playable offline and has LAN support.
some features of the project include -- 

1. LAN auto detect for playing over the same network, direct IP settings for controlling which IP to host the servers on, AND a single config to control all IP's inlcuding Postgre, Redis, and the PR3 servers. Some options in the config include auto mode (auto detect whos host or client), client mode (just launches the game) and host mode (launches the game and servers). Configurable beacon and ping timeout for auto mode on how long to send packets before starting the game as host and more!
 
2.  Account Creation / registration which includes ADMIN for newly created accounts (or a script which turns it off). Give yourself a username, password, and any email (123@example.com) and log in as ADMIN, no tweaking nessesary. Full chat system and functional ADMIN commands- give yourself ANY hat or body part! Want to level up? no problem, just give yourself more EXP :D

3. Complete Campaign system! Includes all the levels from Classic Campaign, Reborn Campaign, and Reborn Campaign 2! Win all the prizes that the campaigns offer, race against your own time and even rank up! Yes you read that right.. the singelplayer campaign now allows you to rank up as you play :). The EXP rate is even set higher so that you can rank up and feel the progression as you beat the campaigns.
  
4. Multiplayer system - Compete, fight, and race your way to victory with your friends- AND win prizes as you do! Includes increased EXP for a more balanced ranking system when its just a couple friends playing. Multiplayer includes over 5000 levels from the best / all time best levels so you arent limited to just the campaign levels. All of the best levels are under a seperate user called BestCampaign and can be played with your friends.

5. Fully functional (or as tested) level, block, and stamp editor. Create, save, and publish your levels! Want to edit one of the campaign or other levels? Just log in as one of the users. Password for all Campaign users is set to 11111111. Included are a few scripts within the data folder which allow for renaming the campaign users if you so desire.

6. Lastly but not least is the abillity to import levels! As a seperate download link ill provide the swf which allows you to download levels as .xml files which can then later be used for importing, playing and the archiving of your levels! Included will be a script and a readme which explains how to import your own levels. The only thing needed is ruffle to load the swf for downloading, or rename the swf in /data/Server/PlatformRacing3.Web/wwwroot and drag and drop the level downloader swf to the same folder and rename to "Singleplayer_2024.swf" and then you can use the preloader to launch the swf. The import script for importing levels will be utilizing the BestCampaign user for importing your levels. The import script CAN be modified to adjust the username BestCampaign but requires a little technical work of changing all references in the script from BestCampaign to your desired username, but its probably easier to just use the included scripts to change the username.

Included in the Releases tab are two appimages and the swf downloader. One appimage is bundled with .net 8 runtime, wine, and the windows servers with the added name (WindowsServer). Both are added for compatibility, if you are having issues with one, try the other. The second appimage is the one youll probably use the most as that seems to be very quick, responsive, and seamless so far. It is bundled with wine (for the preloader), and the servers are all linux! Postgre, Redis, Web and Game servers are all bundled into the appimage. The Appimages are standalone and require no extra setup (hopefully). Just click and play.. or add to steam as a non steam game and play that way. Ill probably include my steam artwork for anyone else who wants to use it. I have yet to make a standalone project for windows but it CAN be done. All the scripts included in the appimage upon extraction include the entire base system for PR3. The create_tables_fixed.sql is just a single script that when imported into postgre will set everything up (minus the levels). The only thing you need installed is postgre and redis, everything else can be manually set up with the postgre script and the level import scripts.

A lot of work went into the project and everything for the most part seems fully functional. If Joni ever decides to make his postgre scripts public one day for offline play, then his project will be superior.. but until then this will do :).
